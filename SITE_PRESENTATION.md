# Презентація проєкту: Nexus Platform

## 1. Мета та призначення проєкту

**Nexus Platform** — це внутрішня веб-платформа, розроблена для команди, що займається арбітражем трафіку. Основна мета платформи — централізувати, автоматизувати та оптимізувати всі ключові аспекти роботи команди:

*   **Управління рекламними акаунтами:** відстеження статусів, аналіз ефективності та підготовка акаунтів до роботи ("фармінг").
*   **Запуск та аналіз рекламних кампаній:** управління бюджетами, креативами та аналіз ROI.
*   **Командна взаємодія:** централізоване управління завданнями, витратами та користувачами.
*   **Інтеграція з Airtable:** використання Airtable як гнучкої та потужної бази даних для зберігання всієї робочої інформації.

## 2. Технічний стек

*   **Фреймворк:** Next.js (React)
*   **Мова:** TypeScript
*   **Стилізація:** Tailwind CSS
*   **База даних:** Airtable
*   **Архітектура:** Клієнт-серверна архітектура на базі Next.js API Routes.

## 3. Реалізований функціонал

На даний момент платформа має наступний готовий функціонал:

### 3.1. Головний дашборд (`/`)

*   **Ключові показники:** Відображення основних метрик ефективності команди, таких як загальний дохід, кількість активних акаунтів, активних кампаній та середній ROI.
*   **Швидкі дії:** Зручні посилання для швидкого переходу до створення нових акаунтів, кампаній, перегляду аналітики та звітів.
*   **Остання активність:** Відображення списків останніх доданих акаунтів та запущених кампаній.

### 3.2. Управління акаунтами (`/accounts`)

*   **Відображення акаунтів:** Сторінка для перегляду всіх рекламних акаунтів у двох режимах: картки та список.
*   **Статистика:** Детальна статистика за статусами акаунтів (активні, заблоковані, в роботі, готові і т.д.).
*   **Фільтрація та пошук:** Потужні інструменти для швидкого пошуку та фільтрації акаунтів за назвою, email, платформою, статусом та іншими параметрами.

### 3.3. Інтеграція з Airtable

*   **Централізоване сховище:** Вся інформація (акаунти, кампанії, витрати, користувачі) зберігається в Airtable.
*   **API для даних:** Створено набір внутрішніх API-ендпоінтів (`/api/airtable/*`) для безпечного отримання даних з Airtable.
*   **Динамічне оновлення:** Додаток завантажує дані безпосередньо з Airtable при завантаженні, що гарантує актуальність інформації.

## 4. Останні виправлення та оптимізації

В ході останньої сесії ми виконали значну роботу по стабілізації та оптимізації платформи:

1.  **Усунення тестових даних:** Повністю видалено всі "заглушки", тестові дані та залежність від `localStorage`. Платформа тепер працює виключно з реальними даними з Airtable.
2.  **Вирішення проблеми "вічного завантаження":** Перероблено `DataProvider` для прямого асинхронного завантаження даних з Airtable, що усунуло блокування рендерингу.
3.  **Виправлення помилки React Hooks:** Оптимізовано порядок виклику хуків у компоненті `AccountsManager`, що виправило помилку "Rendered more hooks than during the previous render".
4.  **Оптимізація рендерингу:** Покращено логіку розрахунку статистики на сторінці акаунтів, що зменшило кількість повторних обчислень.
5.  **Налаштування архітектури провайдерів:** Виправлено ієрархію React Context провайдерів для забезпечення коректної ініціалізації даних про користувача.

## 5. Поточний статус та наступні кроки

**Поточний статус:** Платформа знаходиться в стабільному стані. Ядро системи, що відповідає за отримання та відображення даних з Airtable, працює коректно. Головний дашборд та розділ управління акаунтами повністю функціональні.

**Наступні кроки:**

1.  **Тестування розділів "Кампанії" та "Звіти":** Провести аналогічну перевірку та оптимізацію коду для цих розділів.
2.  **Реалізація логіки CRUD:** Додати функціонал для створення, редагування та видалення записів (акаунтів, кампаній) безпосередньо з інтерфейсу платформи.
3.  **Підготовка до бета-тестування:** Після завершення тестування основного функціоналу, підготувати платформу до запуску для першої групи користувачів.
